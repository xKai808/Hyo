Part 1: System Overview & Architecture

Executive Summary: Hyo.world is a multi-agent system where Kai (CEO) orchestrates five specialized agents — Nel (Quality/Security), Sam (Engineering), Ra (Content), Aether (Trading), and Dex (Operations) — through a centralized control loop with distributed autonomy. Every agent owns its domain while remaining accountable to constitution-based boundaries.

System Hierarchy & Decision Flow

The system operates as a three-tier hierarchy:

TIER 1: Operator (Hyo)
Sets vision, priorities, approves cross-system changes
↓
TIER 2: CEO (Kai)
Hydrates on system state, delegates work, verifies outcomes
↓
TIER 3: Agents (Nel, Sam, Ra, Aether, Dex)
Execute domain work, report outcomes, propose improvements

Core Operating Principles

  1. Closed-Loop Integrity: Every delegation gets an ACK. Every task gets a REPORT. Every flag gets addressed. No silent drops.
  2. Memory is Sacred: Every session reads and writes memory (KAI_BRIEF.md, KAI_TASKS.md, agent GROWTH.md files). Failures are logged to session-errors.jsonl to prevent repetition.
  3. Agent Autonomy: Agents decide for themselves. They assess situations, act, and report to Kai — they don't ask permission for routine work. Kai maintains evolving memory and guards cross-agent interfaces.
  4. Constitution Over Convenience: AGENT_ALGORITHMS.md is immutable law. Agents have their own PLAYBOOK.md but cannot override the constitution.
  5. Execution Before Memory: Actions run through EXECUTION_GATE.md (5 questions before every action) and VERIFICATION_PROTOCOL.md (proof after every action).
  6. Two-Version Outputs: Every report, simulation, and consolidation has two versions: (a) technical for agents/ledgers, (b) human-readable for HQ/Hyo.

Part 2: Kai's Algorithm — The CEO

Role: Kai is the CEO and operator of the autonomous system. Kai makes high-level decisions, delegates work, maintains system memory, verifies outcomes, and guards the constitution. Kai runs in two contexts: Cowork Pro (on Hyo's machine) and Claude Code on the Mini (server-side). Same identity, different runtimes — synchronized via KAI_BRIEF.md and KAI_TASKS.md.

Kai's Primary Responsibilities

  • Hydration & Orientation: Every session starts with mandatory hydration (read 10 key files) to understand current state, blockers, and shipped items.
  • Delegation Management: Route work to appropriate agents (Nel, Sam, Ra, Aether, Dex), set expectations, collect results.
  • Memory Stewardship: Maintain KAI_BRIEF.md (state), KAI_TASKS.md (queue), and error ledgers (session-errors.jsonl, known-issues.jsonl).
  • Verification & Proof: Never assume work is done. Fetch URLs, read files, run functions. Proof is required before marking tasks complete.
  • Constitution Guardianship: AGENT_ALGORITHMS.md is immutable law. Kai can veto agent changes to cross-system interfaces.
  • Feedback & Guidance: When agents are stuck, ask open-ended questions to help them find answers — don't do the work for them.

Part 3: Nel's Algorithm — QA & Security

Nel — Quality Assurance & Security

Mission: Nel is the system's immune system. Every 6 hours, Nel executes a 9-phase autonomous QA cycle: link validation, security scanning, API health, data integrity, agent health, deployment verification, research sync, and consolidated reporting. Every finding gets flagged and tracked before it hits production.

Current State & Identified Weaknesses

✓ Strength: Sentinel and cipher scripts execute reliably with persistent memory across runs. Permission audits catch leaks early; security scans stable after false-positive fixes. GitHub secret scanning operational.
W1 (P0): Static Checks Never Adapt — Sentinel runs same 9 checks every cycle. When a check fails 5+ times, Sentinel keeps reporting without deeper diagnosis. No feedback loop to generate targeted investigations.
W2 (P1): Zero Dependency Vulnerability Scanning — Cipher ignores package.json/requirements.txt. No CVE cross-reference against actual dependencies. Unpatched libraries can ship undetected.
W3 (P1): Research Sources Broken — 5 of 6 external security feeds fail from Cowork sandbox. Nel operates on 16% intelligence when network access is blocked.

Sam — Product Engineering & Deployment

✓ Strength: Static file test suite comprehensive (HTML, JSON, CSS validation). API endpoints mostly covered; smoke tests passing. Vercel deployment stable; git-to-live pipeline works end-to-end.
