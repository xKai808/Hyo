## AGENT-BY-AGENT AUDIT

### 1. Kai (CEO)

**Decision flow:** Session start → hydrate (read 9 files) → delegation checklist → execute or delegate → communicate → shutdown (update memory).

**Bottlenecks found:**
- **B1: Git push requires Hyo.** Every deploy cycle stops at `git push`. Kai edits files, commits locally, then waits for Hyo to physically type a command.
  - **Fix:** MCP server (BUILT, awaiting install). Once connected, Kai calls `git_push` tool directly.
  - **Automation level after fix:** Full. Zero Hyo involvement for deploys.
  - **Status:** Built. Awaiting Hyo to run `npm install` + Tailscale setup.

- **B2: Hydration reads 9 files sequentially.** Context window fills before work starts.
  - **Fix:** Build a `kai hydrate` command that concatenates the 9 files into a single briefing document with a timestamp and diff-from-last-session summary.
  - **Automation:** Script it. `kai hydrate > /tmp/kai-briefing.md` — one read instead of nine.
  - **Owner:** Sam. **Priority:** P2. **Task created.**

- **B3: No automated end-of-session memory write.** If session crashes or compresses, KAI_BRIEF and KAI_TASKS may not get updated.
  - **Fix:** `kai save` already exists but must be called manually. Wire it into a Cowork hook or schedule it every 30 min during active sessions.
  - **Automation:** Create scheduled task `kai-context-save` that runs every 30 min, checks if files have changed since last save, and runs `kai save` if yes.
  - **Owner:** Sam. **Priority:** P1. **Task created.**

### 2. Nel (QA/Security)

**Decision flow:** Scheduled run → 10-phase audit (sentinel, cipher, stale files, permissions, large files, symlinks, scripts, manifests, test coverage, repo size) → report → push to HQ.

**Current scheduled tasks:** `sentinel-hyo-daily` (04:04 MT), `cipher-hyo-hourly`, `hq-ops-sync` (every 4h), `nightly-per-project-consolidation` (23:50 MT).

**Bottlenecks found:**
- **B4: Nel reports issues but doesn't fix them.** Sentinel finds "hyo: 1 failed", "aether: 2 failed" — but nothing automatically files a task or triggers a fix. The findings sit in a .md report until a human reads it.
  - **Fix:** Nel's runner should parse its own output. If sentinel fails > 0 for any project, auto-dispatch a flag: `dispatch flag nel P2 "sentinel: <project> has N failures"`. This already exists in the algorithm but **nel.sh doesn't call dispatch.sh**.
  - **Automation:** Add `dispatch flag` calls to nel.sh after each phase that produces failures.
  - **Owner:** Sam (code change to nel.sh). **Priority:** P1. **Task created.**

- **B5: Simulation failures not auto-escalated.** Last sim: 24 pass, 2 fail (`nel:exit-1`, `ra:exit-2`). These failures are logged to `simulation-outcomes.jsonl` but **nothing reads that file and acts on it**. The nightly reprogramming cycle is designed to do this but isn't implemented as a script.
  - **Fix:** Build `dispatch simulate-review` — a post-simulation step that reads the outcome, checks for failures, and auto-creates tasks for each.
  - **Automation:** Append to the nightly simulation scheduled task: after `dispatch simulate`, run `dispatch simulate-review`.
  - **Owner:** Sam. **Priority:** P1. **Task created.**

- **B6: Cipher runs hourly but has nothing to find.** 51 runs, 0 verified leaks lifetime. The only findings are "gitleaks not installed" (P2, environmental). Running every hour burns compute for no signal.
  - **Fix:** Reduce cipher to daily. Or: install gitleaks/trufflehog so cipher actually does meaningful scanning.
  - **Automation:** Change `cipher-hyo-hourly` cron from `0 * * * *` to `0 4 * * *` (daily at 04:00, right before sentinel). Or keep hourly but only after Hyo installs the scanners.
  - **Owner:** Kai (schedule change) + Hyo (brew install). **Priority:** P3. **Task created.**

### 3. Sam (Engineering)

**Decision flow:** Receive delegation → ACK → search codebase → make changes → run tests → report. Autonomous: scheduled test runs.

**Bottlenecks found:**
- **B7: Sam's test suite can't test API endpoints from sandbox.** 3/16 tests always fail (API egress blocked). These are marked "sandbox-expected" but it means **API regressions are never caught automatically**.
  - **Fix:** Once MCP server is running, add a `kai sam test-api` command that runs on the Mini (full network). Schedule it after every deploy.
  - **Automation:** Add a post-deploy hook: after `git push` succeeds, run `sam.sh test-api` via MCP. If any test fails, auto-flag.
  - **Owner:** Sam. **Priority:** P1 (after MCP is live). **Task created.**

- **B8: No auto-deploy after commit.** `git push` triggers Vercel auto-deploy, but the push itself is manual. Even with MCP, someone has to invoke it.
  - **Fix:** `kai gitwatch` already exists (fswatch on website/). But it runs on the Mini as `nohup kai watch &`. If it dies, deploys stop. No restart logic.
  - **Automation:** Convert `watch-deploy.sh` to a launchd agent with `KeepAlive: true`. Dies → auto-restarts. Create the plist.
  - **Owner:** Sam. **Priority:** P2. **Task created.**

- **B9: website/ vs agents/sam/website/ divergence.** Two copies of the same files. `website/` is git-tracked, `agents/sam/website/` is where Sam's runner references. Edits to one don't propagate to the other.
  - **Fix:** Delete `agents/sam/website/`, symlink it to `website/`. Or vice versa — but `website/` must be the git-tracked real directory because Vercel deploys from it.
  - **Automation:** One-time fix. Then add a Nel check: if both directories exist and aren't symlinked, flag it.
  - **Owner:** Sam (fix) + Nel (monitor). **Priority:** P1. **Task created.**

### 4. Ra (Newsletter/Research)

**Decision flow:** Pipeline: gather.py → ra_context.py → synthesize.py → render.py → ra_archive.py. Autonomous: scheduled at 03:00 MT via aurora-hyo-daily.

**Bottlenecks found:**
- **B10: Aurora runs in Cowork sandbox but can't reach sources.** Every external source (Reddit, HN, arxiv, GitHub, CoinGecko, ProductHunt) returns 403. The pipeline runs daily at 03:00 but **produces nothing** because it can't fetch data.
  - **Fix:** This is the same root cause as B1. Aurora must run on the Mini via launchd, not Cowork scheduled tasks.
  - **Automation:** Already spec'd in KAI_TASKS P0: "Migrate aurora off Cowork scheduled-task sandbox onto Mini launchd." Create the plist, Hyo runs `launchctl bootstrap`.
  - **Owner:** Sam (create plist) + Hyo (install). **Priority:** P0. **Task created.** (Already exists but hasn't shipped.)

- **B11: subscribers.jsonl doesn't exist.** Ra health check warns "subscribers.jsonl missing". Aurora Public can't send emails without subscriber data.
  - **Fix:** Create the file with at least Hyo's email as the first subscriber. Wire `/api/aurora-subscribe` to persist to Vercel KV (already in KAI_TASKS P1).
  - **Automation:** `/api/aurora-subscribe` exists but logs to Vercel function logs (ephemeral). Migration to KV is the automation fix.
  - **Owner:** Sam. **Priority:** P1. **Task created** (already exists in KAI_TASKS).

- **B12: No newsletter produced today.** Ra's report says "No newsletter for today (2026-04-12)". This is a direct consequence of B10 — aurora can't gather.
  - **Fix:** Same as B10. Migrate to Mini launchd.
  - **Automation:** Once on Mini, the cron handles it. Add a sentinel check: "if today's newsletter doesn't exist by 06:00 MT, flag P1."
  - **Owner:** Nel (add sentinel check). **Priority:** P1. **Task created.**

### 5. Aetherbot (Trading)

**Decision flow:** Scheduled every 15 min → check for Monday reset → record trades → update metrics JSON → push to HQ API.

**Bottlenecks found:**
- **B13: No exchange data source connected.** Aetherbot records trades manually via `kai trade '{...}'`. There's no automatic feed from an exchange.
  - **Fix:** Add CCXT integration to aether.sh. Aether reads open positions and closed trades from the exchange API, computes PNL delta, records automatically.
  - **Automation:** Once Hyo provides exchange API keys (read-only), aether.sh runs in `ccxt` mode and pulls every 15 min.
  - **Owner:** Sam (build ccxt integration) + Hyo (provide API keys). **Priority:** P1. **Task created.**

- **B14: Launchd plist not installed.** The 15-min cycle only works when `aether.sh` is scheduled. Plist is built but not loaded.
  - **Fix:** Hyo runs `launchctl bootstrap`.
  - **Owner:** Hyo. **Priority:** P0. **Task created.**

## SYSTEMIC FINDING

The system is good at **detecting** problems but bad at **acting** on them. Reports get generated. Logs get written. JSONL entries get appended. But nothing reads those outputs and creates tasks or triggers fixes automatically.

The fix is the **Automation Gate** — now wired into AGENT_ALGORITHMS.md for every agent, every task. Before and after every piece of work, the system asks: "Can this be automated? Is there a bottleneck? How do we remove it?"

This isn't a one-time audit. It's now a permanent part of the execution loop.
