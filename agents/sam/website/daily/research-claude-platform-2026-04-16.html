Executive Summary

Anthropic's Claude platform has evolved into the most capable AI system for agent-based architectures. For hyo.world's multi-agent system (Kai + 5 sub-agents), Claude offers significant advantages over our current mixed GPT-4o/Claude Code approach.

Key Recommendation: Migrate to Claude Agent SDK (Python) as the unified backend, enable prompt caching for 50% cost reduction, and fix the launchd authentication issue via macOS Keychain token storage.

Why Claude Stands Out

  • 200K Context Window: Claude holds an entire novel of context — critical for agents reading full codebases, research databases, and system state
  • Cost-Effective Stratification: Opus for complex reasoning (CEO tasks), Sonnet for production agents, Haiku for operational checks — matching capability to cost
  • Prompt Caching: 90% discount on repeated context — our agents read the same PLAYBOOKs and AGENT_ALGORITHMS.md every cycle
  • Agent SDK Maturity: Purpose-built for multi-agent systems, beats OpenAI's Assistants API for orchestration and tool use
  • Extended Thinking: Show reasoning chains before final answers — critical for explaining trading decisions and complex analyses

2. Key Capabilities

Multi-Modal (Vision)

Claude can see and analyze images directly — screenshots, charts, documents, diagrams.

  • For Aether: Claude could analyze trading charts visually, not just from log text
  • For Nel: Screenshot-verify that HQ renders correctly across browsers and devices
  • For Sam: Analyze build output visualizations, error dashboards, UI screenshots

Tool Use (Function Calling)

Claude can call external tools/functions during a conversation. Define tools with JSON schemas, Claude decides when to call them and with what arguments.

This is how agents interact with the real world — reading files, making API calls, querying databases, triggering deployments.

Computer Use (Beta)

Claude can control a computer — take screenshots, click, type, scroll. Currently in beta but functional.

  • Applications: Browser-based trading platforms, web app testing, UI verification
  • For Aether: Monitor Kalshi trading UI visually, detect anomalies in real-time
  • For Nel: Full browser testing without manual test scripts

Prompt Caching

When you send the same context repeatedly, Anthropic caches it. Subsequent requests using the same cached prefix cost 90% less for input tokens.

High-Impact for Hyo: Our agents read the same PLAYBOOK.md and AGENT_ALGORITHMS.md every cycle. With caching, this context is cached on the first read and subsequent reads cost 1/10th the price. Potential cost reduction: 40-50% for all agent operations.

Batch API

Submit up to 10,000 requests at once for 50% discount, results within 24 hours.

  • Perfect For: Nightly tasks — consolidation analysis, bulk report generation, research synthesis
  • Not Suitable For: Real-time work (trading decisions, immediate alerts)

Extended Thinking

Claude can "think" before responding — showing its reasoning chain. Useful for complex decisions where you want to see the logic, not just the answer.

  • For Aether: Understanding WHY Claude recommends a trade, not just what. Critical for risk management
  • For Kai: CEO-level reasoning visible and auditable

Structured Outputs

Force Claude to respond in exact JSON schemas. Guarantees parseable output for agent-to-agent communication.

No more regex parsing of freeform text. Agent communication becomes deterministic and reliable.

3. Claude Code (CLI Tool)

What It Does

A command-line tool that gives Claude direct access to your terminal, filesystem, and development environment. It can read/write files, run shell commands, manage git, install packages — essentially a developer copilot.

The Authentication Blocker

Claude Code authenticates via an OAuth flow that opens a browser. In a launchd daemon context (no GUI, no browser), this fails. This is why our agents can't use Claude Code as their backend.

Solution Options

Option 1: API Key Direct

Set ANTHROPIC_API_KEY environment variable in the launchd plist. Claude Code will use it without OAuth.

  • Pros: Simple, immediate, no changes needed
  • Cons: Standard API pricing, no special CLI benefits

Option 2: macOS Keychain

Store the API key in Keychain, retrieve it in the plist's EnvironmentVariables:

ANTHROPIC_API_KEY=$(security find-generic-password -w -s "hyo-anthropic-key" -a "kai")
  • Pros: Secure, no plaintext in plist, elegant
  • Cons: Requires initial Keychain setup

Option 3: Agent SDK (Recommended) Best Path

Skip Claude Code entirely for daemon agents. Use the Python Agent SDK which authenticates via API key natively. No browser needed.

  • Pros: Purpose-built for multi-agent systems, better cost control, no OAuth complexity
  • Cons: Requires Python orchestrator (worth it)

MCP (Model Context Protocol) Integration

Claude Code can connect to MCP servers — plugins that give Claude access to external tools and data.

Custom MCP Opportunities for Hyo:

  • Kalshi Trading API: Aether gets direct market data access
  • Email (Resend): Ra sends newsletters directly
  • GitHub Operations: Sam deploys without manual git push

4. Agent SDK — The Recommended Architecture

What It Is

A Python SDK for building multi-agent systems. Each agent is a Claude instance with its own tools, instructions, and memory. Agents can delegate to sub-agents and coordinate work.

Why This Is Right for Hyo

Our current agents are bash scripts with template output — no real AI reasoning. The Agent SDK replaces this with actual Claude-powered reasoning at each synthesis point:

Agent Currently With Agent SDK
Nel (QA) Runs checks, prints pass/fail Analyzes results, identifies root causes, suggests fixes
Sam (Engineering) Runs builds, reports status Analyzes failures, proposes code changes, optimizes
Ra (Newsletter) Calls OpenAI for synthesis Uses Claude with prompt caching (50% cheaper), better quality
Aether (Trading) Calls GPT-4o for log analysis Uses Claude with extended thinking for deeper strategy analysis
Dex (Ops) Counts stale files, flags issues Analyzes operational patterns, predicts problems before they happen

Proposed Architecture

Python Orchestrator (runs via launchd)
├── Kai Agent (Opus — CEO reasoning)
├── Nel Agent (Sonnet — QA analysis)
├── Sam Agent (Sonnet — engineering)
├── Ra Agent (Sonnet — newsletter synthesis)
├── Aether Agent (Opus — trading analysis)
├── Dex Agent (Haiku — operational checks)
├── Shared Tools (file read/write, git, dispatch)
└── Memory Layer (JSONL ledgers, ACTIVE.md files)

Cost Estimate

With prompt caching and model stratification:

  • Kai (Opus, ~10 calls/day): ~$0.50/day
  • 4 agents (Sonnet, ~20 calls each/day): ~$1.20/day
  • Dex (Haiku, ~50 calls/day): ~$0.10/day
  • Total: ~$1.80/day (~$54/month)
Comparison: Current GPT-4o calls + Claude Code usage likely exceeds $100/month. Agent SDK with caching is cheaper AND more capable.

5. MCP (Model Context Protocol)

What It Is

An open protocol for connecting AI models to external data and tools. Think of it as a USB standard for AI — any tool that speaks MCP can be plugged into any model that supports MCP.

Official MCP Servers (Maintained by Anthropic)

  • Filesystem access
  • GitHub & GitLab
  • Google Drive
  • Slack
  • PostgreSQL & SQLite
  • Puppeteer (browser automation)
  • And more...

Opportunity for Hyo: Custom MCP Servers

Build these immediately:
  • Kalshi API Server: Aether gets direct market data access, order placement, position monitoring
  • Vercel API Server: Sam deploys and monitors without manual git push
  • Email (Resend) Server: Ra sends newsletters directly from Claude
  • Security Scanner Server: Nel runs Trivy, OWASP directly in Claude context

Custom MCP servers eliminate the gap between Claude's reasoning and system actions. Aether doesn't reason then ask a human to place a trade — it reasons AND places the trade.
