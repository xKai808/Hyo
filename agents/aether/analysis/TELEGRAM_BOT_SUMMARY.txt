KAI TELEGRAM BOT — PRODUCTION-READY SCRIPT

SCRIPT LOCATION:
  /sessions/bold-kind-cerf/mnt/Kai analysis/kai_telegram.py
  (553 lines, fully tested syntax)

WHAT IT DOES:
  Full operational interface to Kai via Telegram on Mac Mini. Acts as a bridge
  between mobile (Telegram) and desktop infrastructure.

CORE FEATURES:
  ✓ 8 core commands: /status /balance /analysis /bot /issues /tasks /stop /resume
  ✓ Conversational interface: any text → Claude API with session context
  ✓ Auto-alerts: AetherBot heartbeat (20 min check every 30 min)
  ✓ Auto-alerts: new analysis files detected in real-time
  ✓ Never crashes: all exceptions caught and logged
  ✓ Telegram message limit handling (4096 chars with truncation notice)

ARCHITECTURE:
  - Uses only Python 3 built-ins (urllib, json, os, sys, datetime, threading)
  - No external packages required
  - Long-poll Telegram getUpdates API (30s timeout)
  - Background thread for heartbeat/analysis monitoring
  - Direct HTTP calls to Telegram and Claude APIs

ENVIRONMENT VARIABLES (from ~/Documents/Projects/Kai/.env):
  TELEGRAM_BOT_TOKEN     (required) — from @BotFather
  TELEGRAM_CHAT_ID       (required) — your DM chat ID with bot
  ANTHROPIC_API_KEY      (required) — for Claude conversations
  OPENAI_API_KEY         (optional) — for other integrations

FILE OPERATIONS:
  Read (session context):
    - ~/Documents/Projects/Kai/memory/session_brief.md
    - ~/Documents/Projects/Kai/memory/operator_tasks.md
    - ~/Documents/Projects/AetherBot/Logs/*.log (tail for /bot)
    - ~/Documents/Projects/AetherBot/Kai analysis/*.txt (for /analysis, monitoring)

  Write:
    - ~/Documents/Projects/AetherBot/.stop_flag (for /stop command)

DEPLOYMENT OPTIONS:

COMMAND REFERENCE:

  /status   — Extracts Current State, Active Tracking, Open Issues from session_brief
              Compact mobile-friendly format
  
  /balance  — Extracts balance ledger and current balance from session_brief
              Shows full ledger history and current position
  
  /analysis — Reads latest Final_Analysis, Deep_Analysis, or Analysis file
              Shows first 1500 chars of key findings
  
  /bot      — Tails last 20 lines of latest AetherBot log
              Shows current bot activity
  
  /issues   — Extracts Open Issues section from session_brief
              First 5 issues listed
  
  /tasks    — Reads operator_tasks.md and sends full task list
              For task management and prioritization
  
  /stop     — Writes .stop_flag file to halt AetherBot
              Bot checks this flag on each cycle
  
  /resume   — Removes .stop_flag to restart AetherBot
              Cleanly resumes operation
  
  Any text  — Sends to Claude API with session context
              Uses claude-3-5-sonnet-20241022
              Returns operational response (concise, <1000 chars)

AUTO-ALERTS:

  Heartbeat Check (every 30 min):
    - Checks if AetherBot log was modified in last 20 min
    - If not: sends "⚠️ AETHERBOT HEARTBEAT ALERT" message
    - Prevents silent failures

  New Analysis Detection (real-time):
    - Monitors ~/Documents/Projects/AetherBot/Kai analysis/*.txt
    - When new file appears: auto-sends summary (first 800 chars)
    - Notifies of new analysis without manual check

LOGGING:

  All activity printed to stdout with timestamps:
    [HH:MM:SS] Starting Kai Telegram bot (chat_id=...)
    [HH:MM:SS] Alert loop started (background)
    [HH:MM:SS] Message: /status
    [HH:MM:SS] Sent message to XXX: 523 chars

  When run in background:
    tail -f ~/kai_telegram.log

ERROR HANDLING:

  - All network errors caught, logged, and retried
  - Missing files return error message (no crash)
  - API errors logged with status code, bot continues
  - Malformed .env: logs warning, continues with missing keys
  - Timeout on API calls (30s) — handled gracefully

TECHNICAL DETAILS:

  .env Loading:
    - Matches gpt_factcheck.py pattern (line-by-line parsing)
    - Strips ASCII artifacts from pasted keys
    - Falls back to environment variables if .env missing

  Telegram API:
    - Uses getUpdates with offset tracking (no duplicates)
    - sendMessage with JSON encoding (unicode-safe)
    - 30s timeout on all requests
    - Handles HTTP errors and exceptions

  Claude API:
    - Sends with system prompt containing session context
    - Uses claude-3-5-sonnet-20241022 (fast, cost-effective)
    - 800 token limit on responses (mobile-friendly)
    - Context from first 2000 chars of session_brief.md

DEPENDENCIES:

  ✓ Python 3.7+ (only standard library)
  ✗ No external packages needed
  ✗ No telegram-bot library
  ✗ No requests library
